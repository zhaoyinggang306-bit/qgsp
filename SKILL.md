# 情感短视频生成 Skill

## 适用场景

当用户提出以下需求时使用本 Skill：

- 制作情感类、人生感悟类、婚姻情感类短视频
- 输入一个主题，批量生成文案、分镜、字幕和画面提示词
- 使用成年人物形象、风景、城市夜景等电影感画面
- 为 Wan2.2、图生视频或 RunningHub 工作流准备素材
- 输出可直接配中文旁白的完整项目

## 默认规格

- 画幅：9:16
- 分辨率：1080×1920
- 时长：40–60 秒
- 镜头数：7–9 个
- 单镜头：4–7 秒
- 语言：简体中文
- 人物：明确为成年人物，避免幼态
- 风格：真实、克制、电影感、自然光、轻微运镜
- 推荐模板：人物一致性优先使用 `video Z image wan2.2`；纯场景生成可使用 `video wan2.2`

## 执行流程

### 1. 理解主题和受众

从用户输入中提取：

- 核心主题
- 目标受众
- 情绪基调
- 叙事视角
- 男声或女声偏好
- 是否固定人物形象
- 是否需要批量生成

信息缺失时使用默认值，不要频繁追问。

### 2. 生成视频结构

必须包含：

1. 3 秒内的开头钩子
2. 逐步推进的情绪或故事
3. 一个有记忆点的转折或金句
4. 克制的结尾互动

避免空洞鸡汤、过度煽情、虚假承诺和低俗标题。

### 3. 生成中文口播稿

要求：

- 约 180–280 个汉字
- 适合自然朗读
- 短句优先
- 每句话尽量承载一个画面
- 使用停顿标点
- 不出现“镜头一”等制作术语
- 不虚构名人原话

### 4. 拆分分镜

输出 7–9 个镜头，每个镜头包含：

- 编号
- 时长
- 对应旁白
- 场景
- 人物动作
- 镜头运动
- 中文提示词
- 英文提示词
- 负面提示词
- 转场建议

人物一致性要求：

- 首次定义固定人物卡
- 后续镜头重复核心身份特征
- 不随意改变年龄、脸型、发型、服装主色
- 手部动作简单自然
- 避免大幅度奔跑、复杂肢体和夸张口型

### 5. 生成图生视频提示词

提示词按以下顺序组织：

`主体身份 + 环境 + 动作 + 情绪 + 光线 + 构图 + 镜头运动 + 电影质感 + 画幅`

英文提示词应自然描述，不进行机械逐字翻译。

默认负面提示词：

`low quality, blurry face, deformed hands, extra fingers, duplicate person, childlike face, plastic skin, wax figure, exaggerated expression, strong lip movement, flicker, jitter, warped background, text, watermark, logo`

### 6. 生成字幕

输出标准 UTF-8 SRT：

- 每条 8–18 个汉字为宜
- 单条最多两行
- 标点自然
- 时间连续、不重叠
- 与旁白语义一致
- 不把一句话拆得支离破碎

### 7. 输出项目结构

项目至少包含：

- `project.json`
- `script.txt`
- `voiceover.txt`
- `subtitles.srt`
- `storyboard.md`
- `prompts.json`
- `runninghub-notes.md`

用户需要落地文件时，调用本 Skill 附带脚本：

```bash
python scripts/create_project.py --topic "80后，你快乐吗" --output ./output
```

也可以提供 JSON 输入：

```bash
python scripts/create_project.py \
  --input examples/example-input.json \
  --output ./output
```

### 8. RunningHub / Wan2.2 建议

默认建议：

- 工作流：`video Z image wan2.2`
- 比例：9:16
- 输出：1080×1920
- 人物镜头运动幅度：低到中
- CFG 或提示词强度：中等
- 每个镜头单独生成，再剪辑合成
- 不建议一次生成完整 60 秒长视频
- 人物首帧使用同一角色参考图
- 配音和字幕在剪辑阶段统一合成

不要声称已调用 RunningHub、ListenHub 或其他 API，除非确实获得了可用凭证并完成调用。

## 最终回复格式

优先给用户：

1. 视频标题
2. 完整口播文案
3. 分镜摘要
4. 已生成文件或项目包
5. 一句明确的下一步操作

输出要直接、实用，不堆砌概念。
